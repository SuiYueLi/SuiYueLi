本应用可导入导出节庆民俗列表、每年重复日期列表和笔记。

节庆民俗与每年重复日期都按设定的日历年重复。二者区别在于，节庆民俗出现在日历格的夏历位置，取代一般的夏历日期文字；而每年重复日期只在日历格左上角呈现一个字符图标。

节庆民俗和每年重复日期的导入导出使用 JSON 或 CSV 格式，笔记使用 JSON 或 TXT 文本格式。CSV 格式可使用 WPS Office、Libre Office、MS Office 等办公软件的表格功能打开（或导入）进行编辑。JSON 或 CSV 格式也可直接在记事本等文本编辑器中进行编辑。

导入分为合并与替换两种模式，支持导出的相应格式（应用会自动识别）。

导入文件请参照下文各文件字段说明进行编写，或在导出文件基础上修改其中内容。

数据格式的一般注意事项

JSON

  • JSON 文件须为合规 JSON，编码 UTF-8。
  • 所用符号应为半角标点（即通常英文输入法键入者）。
  • 每组同级的最后一项末尾不要出现终末逗号。
  • 省略项用一对不含任何内容的引号代替，排在末尾的项则可完全省略。

CSV

  • 使用半角（英文）逗号作为分隔符，全角（中文）逗号会被识别为内容的一部分。
  • 半角逗号前后的空格一般会被忽略，作为内容的空格应被成对半角双引号包裹。

笔记 - JSON 格式

按岁、花甲积日两级组织笔记。例：

{
  "4723": {
    "1943975": [
      {
        "icon": "✑",
        "biji": "昔自在古，歷建正，作於孟春。於時冰泮發蟄，百草奮興，秭鳺先嗥。物乃歲具，生於東次，順四時，卒于冬分時；雞三號，卒明；撫十二節，卒于丑。",
        "created": 1779697611921,
        "updated": 1779697611
      }
    ]
  }
}
字段类型说明
iconstring字符图标，缺省 ✑
bijistring笔记内容，单条上限 8000 字
creatednumber创建时间戳（毫秒），作为笔记实质ID
updatednumber最后编辑时间戳（秒）

笔记 - TXT 格式

兼容 Markdown 的三级标题。例：

# 4723
## 4723-04-21
### `✑` 昔自在古，歷建正，作於孟春。於
`[1779697611921 | 1779697611]`
昔自在古，歷建正，作於孟春。於時冰泮發蟄，百草奮興，秭鳺先嗥。物乃歲具，生於東次，順四時，卒于冬分時；雞三號，卒明；撫十二節，卒于丑。
行标记说明
#岁（纪年数）
##笔记节历日期（岁-节-日）
###笔记元数据行：反引号内为字符图标，其后为笔记开头片段
`[… | …]`时间戳行：反引号内方括号为创建时间戳（毫秒）和编辑时间戳（秒）
其他行笔记正文内容

三级标题与时间戳行格式详解：

### `图标` 片段
`[created | updated]`
  • `图标` - 反引号包围的单字符图标，缺省 ✑。
  • 片段 - 笔记内容开头片段，截至15字符或第2个回车。
  • `[created | updated]` - 反引号包围，方括号内为时间戳。
  • 导入时识别时间戳，识别失败将被跳过（完成后列出失败行号）。
  • 导入兼容旧格式 ### "片段" `图标 [created | updated]`。

节庆民俗列表

以 .json 格式为例，顶层对象按历法类型分为四个数组：

{
  "JQ": [ ... ],
  "AL": [ ... ],
  "JL": [ ... ],
  "WC": [ ... ]
}

同日期有多项信息者，在详情中罗列；日历格中靠前者优先。

JQ - 基于节气

字段类型必填说明
始行年number可选华夏纪年
节气编号number必填0~27；0=天正冬至，24=夏历年冬至
相对节气日日数number必填-1=前一日，0=当日，1=次日
日历格显示名string必填显示在日历格中
日期详情显示名string必填显示在详情栏中
终行年number可选华夏纪年

例："JQ": [
  ["", 0, 105, ["寒食", "寒食节"]],
  ["", 24,  0, ["冬至", "冬至节"]]
],

AL - 基于夏历

字段类型必填说明
始行年number可选华夏纪年，缺省太初元年
月number必填1~12
日number必填0=1号前一日，1=1号
日历格显示名string必填
日期详情显示名string必填
终行年number可选华夏纪年

例：["", 1, 0, ["除夕", "除夕夜"]]

JL - 基于节历

格式与夏历相同，年份用华夏纪年。

WC - 基于西历

格式与夏历相同，年份用西元纪年。

例： [1925, 3, 8, ["妇女节", "国际劳动妇女节"]]

每年重复日期列表

以 .json 格式为例，顶层对象按历法类型分为四个数组：

{
  "WC": [ ... ],
  "JL": [ ... ],
  "JQ": [ ... ],
  "AL": [ ... ]
}

同日期有多项信息者，在详情中罗列。日历格显示字符图标，靠后者优先，缺省字符图标为 ❀。

WC - 基于西历

字段类型必填说明
始行年number可选西元纪年
月number必填
日number必填
日期详情显示名string必填
终行年number可选西元纪年
字符图标string可选日历格右上角，缺省 ❀

例： [2003, 6, 7, "普通高等学校招生全国统一考试", "", "✍"]

JL - 基于节历

格式与西历相同，年份用华夏纪年。

JQ - 基于节气

格式与西历类似，年份用华夏纪年；节气编号及相对日数参考节庆民俗部分。

AL - 基于夏历

格式与西历相同，年份用华夏纪年。

例： [4700, 1, 1, "张三生日"]

导入校验

  • 笔记合并导入时将检测同日笔记中创建时间戳相同的项，冲突时弹出列表供处理。
  • JSON 文件须符合其格式规范。
  • 每条记录中，未标注"可选"的字段不可省略。
  • 节庆民俗列表 JQ、AL、JL、WC 四个数组均可省略。
  • 节庆民俗的显示名应为 [日历格显示名, 日期详情显示名] 二元素数组。
  • 每年重复日列表 WC、JL、JQ、AL 四个数组均可省略。
  • 程序未校验年、月、日、节气字段的数字有效性，导入异常数字将造成未预期的结果，严重时可能需清除浏览器缓存数据才能重新使用。
