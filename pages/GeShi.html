本应用可导入导出节庆民俗列表、每年重复日期列表和笔记。

节庆民俗与每年重复日期都按设定的日历年重复。二者区别在于，节庆民俗出现在日历格的夏历位置，取代一般的夏历日期；而每年重复日期只在日历格左上角呈现一个字符图标。

导入导出文件使用JSON格式，其中笔记也可使用文本格式。对于不熟悉JSON格式的用户而言，可在导出文件基础上修改其中内容，得到导入文件。

导入分为合并与替换两种模式。

JSON格式的一般注意事项

  • JSON 文件须为合规 JSON，编码 UTF-8。
  • 所用符号应为半角标点（即通常英文输入法键入者）。
  • 每组同级的最后一项末尾不要出现终末逗号。
  • 省略项用一对不含任何内容的引号代替，排在末尾的项则可完全省略。

节庆民俗列表

文件为 .json 格式，顶层对象包含四个数组：

{
  "JQ": [ ... ],
  "AL": [ ... ],
  "JL": [ ... ],
  "WC": [ ... ]
}

同日期有多项信息者，在详情中罗列；日历格中靠前者优先。

JQ - 基于节气

字段类型说明
节气编号number0~27，0=天正冬至，24=夏历年冬至
距节气日日数number-1=前一日，0=当日，1=次日
日历格显示名string显示在日历格中
日期详情显示名string显示在详情栏中

例："JQ": [
  [0, 105, ["寒食", "寒食节"]],
  [24,  0, ["冬至", "冬至节"]]
],

AL - 基于夏历

字段类型必填说明
始行年number可选华夏纪年，缺省太初元年
月number必填1~12
日number必填0=1号前一日，1=1号
日历格显示名string必填
日期详情显示名string必填
终行年number可选华夏纪年

例：["", 1, 0, ["除夕", "除夕夜"]]

JL - 基于节历

格式与夏历相同，年份用华夏纪年。

WC - 基于西历

格式与夏历相同，始行年/终行年用西元纪年。

例： [1925, 3, 8, ["妇女节", "国际劳动妇女节"]]

每年重复日期列表

文件为 .json 格式，顶层对象包含三个数组：

{
  "WC": [ ... ],
  "JL": [ ... ],
  "AL": [ ... ]
}

同日期有多项信息者，在详情中罗列；日历格显示最后一个字符图标。缺省字符图标为 ❀。

WC - 基于西历

字段类型必填说明
始行年number可选西元纪年
月number必填
日number必填
日期详情显示名string必填
终行年number可选西元纪年
字符图标string可选日历格右上角，缺省 ❀

例： [2003, 6, 7, "普通高等学校招生全国统一考试", "", "✍"]

JL - 基于节历

格式与西历相同，年份用华夏纪年。

AL - 基于夏历

格式与西历相同，年份用华夏纪年。

例： [4700, 1, 1, "张三生日"]

笔记 - JSON 格式

文件为 .json 格式，按岁分组，按花甲积日值组织当日笔记数组。例：

{
  "4723": {
    "1943975": [
      {
        "icon": "✑",
        "biji": "笔记内容",
        "created": 1779697611921,
        "updated": 1779697611
      }
    ]
  }
}
字段类型说明
iconstring字符图标，缺省 ✑
bijistring笔记内容，上限 3000 字
creatednumber创建时间戳（毫秒），作为笔记实质ID
updatednumber最后编辑时间戳（秒）

笔记 - TXT 格式

文件为 .txt 格式，兼容 Markdown 的三级标题。例：

# 4723
## 4723-04-21
### "昔自在古，歷建正，作於孟春。於" `✑ [1779697611921 | 1779697611]`
昔自在古，歷建正，作於孟春。於時冰泮發蟄，百草奮興，秭鳺先嗥。物乃歲具，生於東次，順四時，卒于冬分時；雞三號，卒明；撫十二節，卒于丑。
行标记说明
#岁（纪年数）
##笔记节历日期（岁-节-日）
###笔记元数据：双引号内笔记开头片段，反引号内为字符图标和时间戳
其他行笔记正文内容

三级标题格式详解：

### "片段" `图标 [created | updated]`
  • "片段" - 笔记内容开头片段，截至15字符或第2个回车。
  • `图标 [created | updated]` - 反引号包围，方括号内为时间戳。
  • 导入时识别时间戳，识别失败将被跳过（完成后列出失败行号）。

导入校验

  • JSON 文件须符合其格式规范。
  • 节庆民俗列表 JQ、AL、JL、WC 四个字段均可省略。
  • 每年重复日列表 WC、JL、AL 三个字段均可省略。
  • 每条记录中，未标注"可选"的字段不可省略。
  • 节庆民俗的显示名应为 [日历格显示名, 日期详情显示名] 二元素数组。
  • 笔记合并导入时将检测同日笔记中创建时间戳相同的项，冲突时弹出列表供处理。
